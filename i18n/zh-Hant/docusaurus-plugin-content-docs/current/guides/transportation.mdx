## 概要

import DocCardList from '@theme/DocCardList';

Overture Maps 交通運輸主題是一組描述全球交通基礎設施和出行規範的特徵和屬性。交通資料包括高速公路、人行道、腳踏車道、鐵路、郵輪路線和大眾運輸。

## 使用案例

您可以將 Overture Maps 交通資料和結構用於：

- **地圖繪製：** 渲染連接道路和路徑的地圖。
- **路由規劃：** 計算從一個地方到另一個地方的最佳路徑。
- **導航：** 生成遵循路線所需的詳細指令。
- **分析：** 包括交通安全分析和災害規劃在內的交通相關分析。
- **地理編碼：** 獲取街道交叉口的坐標（地理編碼）或特定坐標附近的街道交叉口（反向地理編碼）。

## 核心概念

- **形狀和連通性。** 交通運輸主題結構捕捉交通網路的形狀和連通性，使用 `segment` 和 `connector` 特徵。結構設計允許分段過程促進形狀穩定性，最終促進特徵 ID 的穩定性。請在 [形狀和連通性](/schema/concepts/by-theme/transportation/shape-connectivity) 頁面上閱讀更多訊息。

- **範圍和基於規則的屬性。** `transportation` 主題結構允許為細粒度範圍在子特徵級別指定屬性值。例如：
  - 道路段上的速度限制可能範圍限定於道路幾何的一部分，使用 _幾何範圍限定_。
  - 段的方向性，控制交通可以沿著段幾何流動的方向，可能會根據一天中的不同時間進行變更，使用 _時間範圍限定_。
  - 允許在段上旅行的人員和車輛的類別可以使用 _主觀範圍限定_ 進行控制。
  - 隨著環境條件變化而變化的實際屬性可以使用 _環境範圍限定_ 進行建模。

  在 [範圍和基於規則的屬性](/schema/concepts/scoping-rules) 頁面上閱讀更多這些概念的訊息。

- **道路。** `transportation` 主題結構將任何類型的道路、街道或路徑，包括專用步行和自行車路徑，建模為 `road` 段。道路目前是 `transportation` 結構中最發展的部分。請在 [道路](/schema/concepts/by-theme/transportation/roads) 頁面上閱讀更多有關道路的訊息。

- **旅行結構。** `transportation` 主題支援一種模糊概念稱為旅行結構，可以用作控制範圍和基於規則的屬性的方式。請在 [旅行結構](/schema/concepts/by-theme/transportation/travel-modes) 頁面上閱讀更多有關旅行結構概念及其如何與其他範圍屬性互動的訊息。
