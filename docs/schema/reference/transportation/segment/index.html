Segment

Fields​

NameTypeDescription
idIdA feature ID. This may be an ID associated with the Global Entity Reference System (GERS) if—and-only-if the feature represents an entity that is part of GERS.
bboxbbox (optional)An optional bounding box for the feature
geometrygeometrySegment centerline

Allowed geometry types: LineString
theme"transportation"
type"segment"
versionFeatureVersion
sources[](10)Sources (list, optional)Information about the source data used to assemble the feature.
subtypeSubtypeBroad category of transportation segment.
access_restrictions[](9)AccessRules (list, optional)Rules governing access to this road segment
connectors[](2)list<ConnectorReference> (optional)List of connectors which this segment is physically connected to and their relative location. Each connector is a possible routing decision point, meaning it defines a place along the segment in which there is possibility to transition to other segments which share the same connector.

Minimum length: 2
All items must be unique. (UniqueItemsConstraint)
level_rules[](2)LevelRules (list, optional)Defines the Z-order, i.e. stacking order, of the road segment.
routes[](6)Routes (list, optional)Routes this segment belongs to
subclass_rules[](2)SubclassRules (list, optional)Set of subclasses scoped along segment
names(11)Names (optional)All known names by which the feature is called
class (Road)RoadClass
destinations[] (Road)(10)Destinations (list, optional)Describes objects that can be reached by following a transportation segment in the same way those objects are described on signposts or ground writing that a traveller following the segment would observe in the real world. This allows navigation systems to refer to signs and observable writing that a traveller actually sees.
prohibited_transitions[] (Road)(12)ProhibitedTransitions (list, optional)Rules preventing transitions from this segment to another segment.
road_flags[] (Road)(2)RoadFlags (list, optional)Set of boolean attributes applicable to roads. May be specified either as a single flag array of flag values, or as an array of flag rules.
road_surface[] (Road)(2)Surfaces (list, optional)Physical surface of the road. May either be specified as a single global value for the segment, or as an array of surface rules.
speed_limits[] (Road)(15)SpeedLimits (list, optional)Rules governing speed on this road segment
subclass (Road)Subclass (optional)
width_rules[] (Road)(2)WidthRules (list, optional)Edge-to-edge width of the road modeled by this segment, in meters.

Examples: (1) If this segment models a carriageway without sidewalk, this value represents the edge-to-edge width of the carriageway, inclusive of any shoulder. (2) If this segment models a sidewalk by itself, this value represents the edge-to-edge width of the sidewalk. (3) If this segment models a combined sidewalk and carriageway, this value represents the edge-to-edge width inclusive of sidewalk.
class (Rail)RailClass
rail_flags[] (Rail)(2)RailFlags (list, optional)Set of boolean attributes applicable to railways. May be specified either as a single flag array of flag values, or as an array of flag rules.
